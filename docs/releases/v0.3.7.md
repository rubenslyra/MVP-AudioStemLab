# Release v0.3.7 - Portable Assets With Split CUDA Packages

This release publishes ready-to-download desktop packages through the repository release page.

## Recommended Download

Use the CPU package unless you know the machine has a compatible NVIDIA GPU:

```text
AudioStemLab-v0.3.7-linux-cpu.zip
AudioStemLab-v0.3.7-windows-cpu.zip
AudioStemLab-v0.3.7-macos-cpu.zip
```

Use CUDA only for supported NVIDIA systems:

```text
AudioStemLab-v0.3.7-linux-cuda.zip.part-*
AudioStemLab-v0.3.7-windows-cuda.zip.part-*
```

macOS is CPU-only.

## CUDA Package Notes

GitHub release assets cannot exceed 2 GiB. CUDA packages are larger than that, so they are published as multiple `*.part-*` files.

Download every part for the CUDA package you need, keep them in the same folder, and rebuild the ZIP before extracting it.

Linux or macOS:

```bash
cat AudioStemLab-v0.3.7-linux-cuda.zip.part-* > AudioStemLab-v0.3.7-linux-cuda.zip
```

Windows PowerShell:

```powershell
cmd /c copy /b AudioStemLab-v0.3.7-windows-cuda.zip.part-aa+AudioStemLab-v0.3.7-windows-cuda.zip.part-ab AudioStemLab-v0.3.7-windows-cuda.zip
```

If more Windows CUDA parts are present, include each part in alphabetical order in the `copy /b` command.

## What Changed

- CPU packages are uploaded as normal ZIP files.
- Oversized CUDA packages are split into release assets smaller than GitHub's asset limit.
- Release documentation now explains how to rebuild split CUDA ZIPs.

## End User Requirements

End users do not need to install Python, create a virtual environment, or run `pip`.

Download the package for the operating system, extract it, and open the matching launcher.

## Notes

Demucs model weights may still be downloaded on first use if they are not already available in the user's model cache.
