# Packaging

This project is prepared to ship portable desktop packages for Linux, Windows, and macOS.

End users should receive ZIP packages generated by GitHub Actions. They should not install Python, create a virtual environment, or run package managers.

## Release Artifacts

Expected artifact names:

```text
AudioStemLab-v0.3.8-linux-cpu.zip
AudioStemLab-v0.3.8-windows-cpu.zip
AudioStemLab-v0.3.8-macos-cpu.zip
AudioStemLab-v0.3.8-linux-cuda.zip.part-*
AudioStemLab-v0.3.8-windows-cuda.zip.part-*
```

Each ZIP includes:

- PyInstaller output under `AudioStemLab/`.
- Platform launchers under `launchers/`.
- `README.md`.
- `CHANGELOG.md`.
- `VERSION`.
- `LICENSE`.
- `RELEASE_NOTES.md`.

The current package includes the application runtime. Demucs model weights may still be downloaded on first use if they are not already present in the user's model cache.

CUDA ZIPs can exceed the GitHub release asset limit of 2 GiB. The release workflow splits oversized ZIPs into `*.part-*` assets under 1.9 GiB. Users must download every part for the selected CUDA package and concatenate them before extraction.

## Automated Build

Builds run from:

```text
.github/workflows/release.yml
```

The workflow builds on:

- `ubuntu-latest`
- `windows-latest`
- `macos-latest`

It runs on tags matching:

```text
v*
```

## CPU And CUDA Variants

CPU packages are the default recommendation for testers. They avoid shipping NVIDIA CUDA libraries and should work on a broader range of desktop machines.

CUDA packages are generated separately for Linux and Windows. They are intended only for computers with compatible NVIDIA GPUs and drivers.

macOS builds are CPU-only.

## Manual Build

For contributors:

```bash
python -m pip install --upgrade pip wheel
python -m pip install -r requirements.txt
python -m pip install -r requirements-build.txt
python -m pip install -r requirements-cpu.txt
python -m pip install -r requirements.txt
pyinstaller --clean --noconfirm packaging/AudioStemLab.spec
python scripts/package_release.py --version v0.3.8 --platform linux --acceleration cpu
```

Run the manual build on each target OS to produce the matching portable package.

## Notes

The runtime includes Demucs, PyTorch, torchaudio, and torchcodec. These dependencies are large.

The CPU/CUDA split was added in `v0.3.5` to reduce the default package size and avoid forcing CUDA libraries on machines that cannot use them.

Local Linux validation for `v0.3.4` produced:

```text
dist/: 4.5G
AudioStemLab-v0.3.4-linux.zip: 3.9G
```

The executable was validated to start and exit from the generated PyInstaller package.
